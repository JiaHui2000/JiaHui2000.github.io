数据结构

发表于 2021-05-21 更新于 2022-07-09 阅读次数： 阅读次数： Valine：
记录数据结构相关知识。

第1章 绪论

1.1 基本概念

1.1.1 基本术语

  1. 数据
  2. 数据元素
  3. 数据对象
  4. 数据结构：将关系组织成某种形式的信息表，以便程序进行加工处理；

1.1.2 数据结构三要素

1）逻辑结构

定义：数据元素之间的逻辑关系，与数据的存储无关，独立于计算机。

分类：线性结构、非线性结构。

2）存储结构

定义：数据结构在计算机中的表示（映像），又称物理结构。

内容：数据元素的表示和结构关系的表示。

分类：

  1. 顺序存储
  2. 链式存储
  3. 索引存储
  4. 散列存储

各种存储方式的优缺点是什么？

3）数据的运算

定义：针对逻辑结构的定义，针对存储结构的具体操作。

1.2 抽象数据类型

数据类型：一个值的集合和定义在该集合上的操作的总成。

分类：

  1. 原子类型：bool类型：值为TRUE和FALSE，操作为与或非。
  2. 结构类型：struct坐标。
  3. 抽象数据类型：抽象数据组织以及相关的操作。忽略操作的实现以及在内存中的表示形式。

1.3 算法与算法评价

程序=数据结构+算法

1.3.1 算法的概念

算法algorithm通常指解决问题的方法和过程。是指令的有限序列，每条指令表示一个或多个操作。

特性：

  1. 输入：零个或多个，输入取自于某个特定对象的集合。
  2. 输出：至少有一个。
  3. 有穷性：有穷步，有穷时间。
  4. 确定性：指令清晰无歧义，相同输入相同输出。
  5. 可行性：每个运算步骤都可以精确地执行。

注：程序是算法用某种程序设计语言的具体实现，程序可以不满足算法的有穷性。

1.3.2 算法的复杂度及其表示

设计算法的目标：

  1. 正确性
  2. 可读性
  3. 健壮性
  4. 效率与低存储量需求

时间复杂度

事先预估算法时间开销T(n)与问题规模n的关系。

T(n)=O(f(n))只需要关注高阶部分。

两条规则：

  1. 加法规则
  2. 乘法规则

分析规则：

  1. 顺序执行的代码只会影响常数项可以忽略。
  2. 只需挑循环体中的一个基本操作分析他的执行次数与n之间的关系。
  3. 多层嵌套循环只需关注最深层循环执行次数。

空间复杂度

S(n)定义为算法执行所需的空间，是问题规模n的函数。

S(n)=O(g(n))

程序执行时所需的存储空间包括：本身所用的指令、常数、变量和输入数据外，还需要一些对数据进行操作的工作单元和存储一些为实现计算所需的辅助空间。程序代码所需空间固定，与问题规模无关。只需关注存储空间大小与问题规模相关的变量。

原地工作：指算法所需的辅助空间为常量，O(1)。

函数递归带来的内存开销：

每一层所需要的空间相同：S(n)=O(n) 空间复杂度=递归调用深度

每一层所需要的空间不同：具体分析
